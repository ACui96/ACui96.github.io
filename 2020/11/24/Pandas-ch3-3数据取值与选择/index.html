3.3.1 Series数据选择方法

1. 将Series看作字典

a    0.25
b    0.50
c    0.75
d    1.00
dtype: float64
0.5

字典的表达式和方法

True
Index(['a', 'b', 'c', 'd'], dtype='object')
[('a', 0.25), ('b', 0.5), ('c', 0.75), ('d', 1.0)]

利用字典语法调整数据。字典可以增加新的键，那Series也可以增加新的索引。

a    0.25
b    0.50
c    0.75
d    1.00
e    1.25
dtype: float64

2、将Series看作一维数组

和Numpy数组一样，Series具有索引、掩码、fancy索引等操作

a    0.25
b    0.50
c    0.75
dtype: float64
a    0.25
b    0.50
dtype: float64
b    0.50
c    0.75
dtype: float64
a    0.25
e    1.25
dtype: float64

上述实例中，容易混淆的是切片。显式索引包含最后一个索引，隐式索引不包含最后一个索引。

3、索引器： loc 、iloc 和 ix

容易混淆的情况：

1    a
3    b
5    c
dtype: object
'a'
3    b
5    c
dtype: object

Pandas提供了一些索引器(indexer)属性来作为取值的方法，它们不是Series对象的方法，而是暴露切片接口的属性。

loc属性，表示取值和切片都是显式的：

'a'
1    a
3    b
dtype: object

iloc属性，表示取值和切片都是Python形式的隐式索引 从0开始，左闭右开区间。

'b'
3    b
5    c
dtype: object

第三种属性是ix，是前两种的混合形式，在Series对象中ix等价于标准的[]（Python列表）取值方式。主要用于DataFrame对象。

3.3.2 DataFrame数据选择方法

1、将DataFrame看作字典

area pop
California 423967 38332521
Texas 695662 26448193
New York 141297 19651127
Florida 170312 19552860
Illinois 149995 12882135

两个Series分别构成DataFrame的一列，可以通过队列名进行字典形式的取值获取数据：

California    423967
Texas         695662
New York      141297
Florida       170312
Illinois      149995
Name: area, dtype: int64

同样，可以用属性形式选择纯字符串列名的数据：

California    423967
Texas         695662
New York      141297
Florida       170312
Illinois      149995
Name: area, dtype: int64
True

虽然属性形式很方便，但它并不是通用的。当列明不是纯字符串，或者列名与DataFrame的方法同名，Mamet就布恩那个用属性索引。
例如，DataFrame有一个pop()方法，如果用data.pop不会获取’pop’列，而是显示为方法。

False

另外，要避免用属性形式对列直接赋值(即可以用data[‘pop’] = z, 但不要用data.pop = z)。

用字典形式的语法调整对象：

area pop desity
California 423967 38332521 90.413926
Texas 695662 26448193 38.018740
New York 141297 19651127 139.076746
Florida 170312 19552860 114.806121
Illinois 149995 12882135 85.883763
